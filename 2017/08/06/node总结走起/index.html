node总结

By 王邸 at: 2017-08-06

总结

这一周过的一般般吧。也许刚开始起步就是那么安静的。还是那句话：不以物喜不以己悲。上升到一种状态就很好了。这一周在公司也没有特别重的开发项目。主要还是以学习为主。但是学习了一些和工作无关的东西，所以不是很好。自己动脑想想也就知道。这不是一个好的开始。一定要站在公司角度想想。这周做了什么呢？看标题啊node！！！！这周其实做了挺多东西的。主要还是做了Node + vue + mysql/mongodb，在宿舍走了一波老路node + koa 微信公众号的开发。除了技术上的，其实对时间管理还是已经慢慢开始使用了。这是一个我非常喜欢的。这样过的才有价值了兄弟。

VUE+node+mongodb/mysql

使用了VUE + vue-resource + mongodb，地址传送门,这个地址就是整个任务具体流程。我也就是走通了一遍。并没有什么特别大的问题。之前自学node.js的开发指南时候就是这样的一个类似的demo。使用的是bootrap+express+mongodb。这个小项目目的是走通和数据库的连接数据流程。了解现在常用的库等等

VUE

VUE没什么好说的都没有写vue-router可以搞点事情

node

这里使用了express框架中文文档,node博大精深。还是需要很多知识支撑的。作为一个入流的前端node的知识需要学习的。其他的一些body-parser、fs、mongoose、path的库看看官网知道用这些来做什么就好的。有些已经集成在node里了有些还是package。

experss主要还是使用了路由的功能。路由就好像控球后卫一样，之前学习时候的比喻一直生动形象。把请求分发给不同的接口去处理web服务器的请求。这些请求对应就是需要返回给前端的。其他的一些库会用就行了。

mongodb

参照之前的博客写的mongodb的安装入门教程就能入门啦mongodb。非关系型数据库存储的就是json。引入mongodb插件最好看看mongodb的文档。连接nodeJS还是贼爽的。

mysql

最后使用了mysql去实现一下同样的功能，其实也不是特别的炸吧。不过使用mysql的库的时候,使用连接池的方式连接比较好。因为不能调一次就开关一次数据库，有同伴公司是用连接池的方式实现的。mysql对于大型的数据保存还是比较友好的。但是安装起来还是比较蛋疼的。出现了很多小bug。主要还是需要的mysql安装要安装microsoft visual c++ 2013的32位.一直安装64位的就是不行。原来是需要32位的。好像也没别的。

node + koa 实现微信公众号

koa

koa已经支持ES7的语法格式了，使用的是async函数，但是现在微信这个项目还没有深入下去。这周继续做的是代码风格的事情
所以 这周的总结写的是代码风格的事情。

ngrok

这是内网映射的一个功能。把本地的服务印刷到公网上
