强制类型转换：

目标类型 变量名= (目标类型)(被转换的数据);
一般建议，最好不要随意使用强制类型转换，容易造成数据精度的损失。
1) 表数范围大的数据类型转换为表数范围小的数据类型时，要加上强制转换符，但可能造成精度损失；使用时要格外注意。
2) 有多种类型的数据混合运算时，系统首先自动的将所有数据转换成容量最大的那一种数据类型，然后再进行计算。

汉诺塔问题：
汉诺塔是根据一个传说形成的数学问题：

有三根杆子A，B，C。A杆上有 N 个 (N>1) 穿孔圆盘，盘的尺寸由下到上依次变小。要求按下列规则将所有圆盘移至 C 杆：

每次只能移动一个圆盘；
大盘不能叠在小盘上面。
提示：可将圆盘临时置于 B 杆，也可将从 A 杆移出的圆盘重新移回 A 杆，但都必须遵循上述两条规则。

问：最少要移动多少次？如何移？

分析：
n=1:
A –> C
count = 1

n=2:
A –> B
A –> C
B –> C
count = 3

n=3:
A –> C
A –> B
C –> B
A –> C
B –> A
B –> C
A –> C
count=7;

分析：
假设移动n个圆盘最小需要count(n)次，递归的公式：
count(1) = 1;
count(n) = 2* count(n-1) + 1 （n > 1）; // 两个count(n-1)次 一次是把上面n-1个移动到B 第二次是把n-1个从B移动到C

h(1) = 1;
h(n) = 2h(n-1) + 1;
h(n) + 1 = 2 (h(n-1) + 1);
h(n) + 1 = 2 * 2^(n-1) = 2^n;
h(n) = 2^n - 1;

结论：从以上可以看出，递归解决的不是计算移动多少次的问题。而是要去解决如何去移动的问题

如何移？
定义一个函数表示把n个盘子从begin,通过middle,移动到end上面。
返回值类型: void
参数列表：int n, char begin, char middle, char end

递推公式：
把n-1盘子从begin,经过end,移动到middle
把最大的盘子从begin移动到end.
把n-1盘子从middle,经过begin,移动到end

  2. 子问题的解决方法和大问题一模一样，只是数据规模不一样。
  3. 有终止条件。

f(n) = f(n-1) + n

bem（block element modifier）

这是一种非常格式化的元素的class的命名方式,这种命名方式看起来非常的分明。什么是block 什么是element 什么是修饰词modifier。这个规则最早还和大漠老师有问到。不过还是自己去了解清楚了。是一种很规范的命名方式。接下来还会介绍另外两种。最主要的是统一。这种用法优点就是类看起来比较清晰就像下面这样：

<div class="block">
  <div class="block-head">
    <div class="block-head-navbar"></div>
    <div class="block-head-logo"></div>
  </div>
  <div class="block-body"></div>
  <div class="block-footer"></div>
</div>

而传统的一般我会这样命名

<div class="block">
  <div class="blockHead">
    <div class="narbar"></div>
    <div class="narbarLogo"></div>
  </div>
  <div class="blockBody"></div>
  <div class="blockFooter"></div>
</div>

他这种写法在写css样式表时，能够很清楚的知道控制的是哪个标签。当然缺点也是显而易见的，就是双击并不能全部选中。只能选中某一部分，如果使用下划线“_”来命名也不是特别的友好。对于爬虫等等来说。接下来介绍两个很类似的命名方式。

OOCSS （Object Oriented CSS）and SMACSS（Scalable and Modular Architecture for CSS）

这两个的思想很类似的，就是把CSS抽离出来各司其职。因为css有很多部分有的控制布局，有的控制样式。为了工作更加的高效，搞得都是模块化的思想来做。OOCSS思想是：
结构域外观分离、容器与内容分离。这就是结构的block就写结构外观skin就写外观。容器就好定位。内容就单独设置样式。

另一种SMACSS这是一种更加分离的，我个人也比较喜欢这种样式编写方式，把样式分成五个部分

  • Base（基本），定义元素的默认样式。可以把 Normalize.css 放在这里
  • Layout（布局），可以参考一下 Susy 这种基于列宽而又独立于模块的类名
  • Module（模块），定义模块时应全部使用类名选择器，不要使用元素选择器
  • State（状态），注意要与模块的修饰器区分
  • Theme（皮肤）
    分离出来以上五种，并在相应的部分注释上这五种分类，也会很清楚看出css控制的是哪个。大家开发才会更加高效。这礼拜主要也是修改样式并没有特别多的内容。下礼拜着重回顾element的那些用法。

  3. 好好把前端做好

  5. 其他各种杂事
